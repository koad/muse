# Component Patterns

Reusable UI component patterns and specifications.

## Core Components

- **Typography:** Headings, body text, labels, code blocks
- **Forms:** Inputs, buttons, checkboxes, selects, validation states
- **Cards:** Standard card layout with title, content, actions
- **Navigation:** Navigation patterns, breadcrumbs, tabs
- **Modals/Overlays:** Dialog boxes, dropdowns, popovers
- **Lists:** List layouts, pagination, empty states
- **Feedback:** Alerts, toasts, loading indicators, error states

## Documentation Structure

Each component pattern should include:
- Visual examples (mockups or screenshots)
- Markup structure (semantic HTML)
- CSS classes or custom properties used
- Responsive behavior
- Accessibility notes (ARIA, keyboard navigation)
- Usage guidelines (when to use, constraints)
