## What This Page Is

koad.sh is not a portfolio. It is not a bio. It is not a landing page for a product.

It is the answer to `whoami` — run on a machine you control, printing exactly what matters and nothing else. Sparse. Confident. Terminal-native in feel.

The guiding constraint: **if you would not put it in a README, do not put it on this page.**

## Design Notes

**Background:** `--bg-primary` (#0c0c0e). Same as the rest of the ecosystem. No exceptions.

**Name:** "koad" in lowercase. Not "Koad." Not "KOAD." The casing is intentional — it reads like a username, not a brand name.

**Descriptor:** One or two sentences maximum. Current text: *"Builder of koad:io — persistent infrastructure for AI entities."* Do not expand this. The brevity is the message.

**Link list:** The label/value layout uses a fixed-width label column (5 characters). Values are `--accent` (#7c6aff) at rest, underline on hover. All links open in new tab with `rel="noopener noreferrer"`.

**Entity roster:** Each entity name links to its profile page (`kingofalldata.com/entities/{entity}`). The descriptor is the entity's one-line role — pull from `2026-04-05-entity-descriptions.md`. The roster is a directory, not a pitch. No accents per-entity, no cards, no badges.

**No navigation.** There is no header nav, no footer nav, no back button. This page does not belong to the kingofalldata.com navigation tree — it is a standalone page at a separate domain.

**No Keybase badge widget, no GitHub stars count, no "last updated" timestamp.** Static. Committed. No dynamic elements.

**Footer:** One line, bottom of page, `--text-muted` 12px:
```
koad.sh · keys at canon.koad.sh
```
That's it. The keys reference is the one link worth repeating — it anchors trust without explaining itself.

## Tone

The page should feel like meeting someone at a conference and they hand you a card that has exactly: their name, what they build, where to find them. Nothing more. You already know you want to talk to them because the card told you in four words what they do.

Resist the urge to add context. Resist the urge to explain koad:io. That's what kingofalldata.com is for. This page is a coordinate, not a pitch.

---

## What This Page Is Not

- Not a portfolio ("here are projects I've built")
- Not a bio ("I've been building software since...")
- Not a landing page ("Try koad:io today")
- Not a news feed ("Latest from koad")

Every one of those things belongs somewhere else. This page is the answer to: *who is this person and where do I find them.*
