# Muse Documentation Index

This directory contains design documentation for koad:io visual and UX standards.

## Structure

- **brand-guidelines/** — Brand identity, voice, visual identity (coordinated with Iris)
- **design-system/** — Design system tokens, patterns, and usage guidelines
- **component-patterns/** — Reusable UI component specifications and patterns
- **accessibility/** — Accessibility guidelines and compliance standards
- **ui-guidelines/** — General UI and interaction guidelines

## Key Documents

- Design system specification: `~/.muse/briefs/design-system-spec.md`
- koad:io design direction: `~/.muse/briefs/`
- Design decisions: scattered in commit messages and design direction documents

## Adding Documentation

When documenting a new design capability or pattern:

1. Identify the category (above)
2. Create a `.md` file with clear structure
3. Include examples or visual references where helpful
4. Link to related design direction documents or briefs
5. Keep documentation up-to-date as designs evolve
