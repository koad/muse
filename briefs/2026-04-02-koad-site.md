# UI/UX Brief: kingofalldata.com PWA

**Project:** Sovereign Identity Platform
**Author:** Muse (UI Beauty Entity)
**Date:** 2026-04-02
**Status:** Design Direction Brief

---

## Visual Tone

**Sovereign. Minimal. Identity-forward.**

The interface should feel like a digital passport or identity card — not a SaaS dashboard. Every element serves the user's identity, not the platform's brand. The platform disappears; the person remains.

- **Aesthetic:** Terminal-adjacent minimalism. Think: cryptographic elegance. Clean lines, purposeful spacing, information density that respects the user.
- **Mood:** Trustworthy, sovereign, slightly technical — but approachable. Not cold. Not corporate.
- **Personality:** The confidence of knowing exactly who you are. No marketing fluff on user pages.

---

## Typography Direction

**Primary:** A monospace typeface for keys, technical data, and namespace labels — reinforces the cryptographic/sovereign identity theme.

**Secondary:** A clean sans-serif for body copy, pitches, and UI chrome.

| Role | Font | Weight |
|------|------|--------|
| Namespace labels, keys, technical data | JetBrains Mono or IBM Plex Mono | 400, 500 |
| Headlines, section titles | Inter | 600, 700 |
| Body copy, CTAs, UI chrome | Inter | 400 |

**Scale:** 14px base, 1.5 line-height for body. Compact but readable. Mobile-first sizing — generous touch targets.

## Key Interaction Patterns

### Namespace Profile Pages

- **Hero:** Avatar + display name + username (`.kingofalldata.com`). No platform branding.
- **Navigation:** Tab bar for `/profile`, `/keys`, `/emitters`, `/trust`. Minimal underline indicators.
- **Keys tab:** List of public keys with fingerprint, type, and copy button. Copy-on-click with brief "Copied" feedback.
- **Emitters tab:** Presence indicators. Green dot = online. Timestamps in relative format ("2m ago").
- **Trust tab:** List of trust bonds. Avatar + name + verification status.

**Pattern:** Zero friction. Everything copyable. Key verification visible. Presence ambient but not distracting.

### Root Pitch Pages (Domain Skins)

- **Hero:** Headline → Subheadline → CTA. Single focus. No carousel.
- **Social proof:** Minimal. Maybe 2-3 testimonials or a live counter if appropriate.
- **CTA:** GitHub Sponsors → namespace provisioning. Clear, direct.
- **Below fold:** Features listed without icons or imagery. Text-forward.

**Pattern:** Conversion-focused but not aggressive. The system sells itself.

### MVP Zone

- **Layout:** Sidebar navigation (desktop) / bottom tab bar (mobile).
- **Directory:** Grid of namespace cards. Avatar + name + brief bio. Filterable.
- **Build stream:** Chronological feed. Each entry: entity avatar, action, timestamp.
- **Dark Passenger indicator:** Subtle browser extension status in corner.

### Global Patterns

- **Loading states:** Skeleton screens, not spinners. Skeleton color matches `--bg-tertiary`.
- **Error states:** Inline. Red border + message below input. No modal errors for form validation.
- **Copy feedback:** "Copied" appears adjacent to button, fades after 1.5s.
- **Navigation:** Instant on PWA. Use browser navigation API for back/forward.
- **Offline:** Show cached namespace data with "Offline" banner. Keys remain readable.

## Responsive Strategy

- **Mobile-first:** All layouts start from mobile viewport.
- **Breakpoints:** 
  - `sm: 640px` — two-column grids
  - `md: 768px` — sidebar appears
  - `lg: 1024px` — comfortable reading width for content
- **Touch targets:** Minimum 44x44px for all interactive elements.
- **Namespace pages:** Stack gracefully. Avatar center-aligned on mobile.

---

## Accessibility

- **Contrast:** All text meets WCAG AA (4.5:1 for body, 3:1 for large text).
- **Focus states:** Visible outline using `--accent` color. No focus hiding.
- **Motion:** Reduced motion respected via `prefers-reduced-motion`. Key animations: fade only.
- **Screen readers:** Semantic HTML throughout. Key fingerprints announced as monospace.

---

## Performance Targets

- **First Contentful Paint:** < 1.5s
- **Time to Interactive:** < 3s
- **Lighthouse score:** > 90 on all categories
- **Bundle:** < 150kb initial JS. Lazy-load MVP Zone sections.
- **Images:** WebP with fallbacks. Lazy-load below fold.

---

## Anti-Patterns to Avoid

- ❌ Platform logo on namespace pages
- ❌ "Powered by kingofalldata.com" watermarks
- ❌ Carousels or hero sliders
- ❌ Lightbox modals for primary content
- ❌ Decorative gradients on user-facing pages
- ❌ Heavy drop shadows
- ❌ Sans-serif fonts for technical data (keys, fingerprints)

---

## Implementation Notes for Vulcan

1. **CSS Variables:** All colors via variables in `:root` and `[data-theme="light"]`. Domain skins can override `--accent` on root pitch pages only.
2. **No Tailwind by default:** Custom CSS with variables. Tailwind adds bundle weight and isn't necessary for this scope.
3. **Avatar system:** Support WebP, SVG, and gradient fallbacks. Circular crop with 2px `--border` ring.
4. **Dark Passenger integration:** Reserve a corner position for DP status indicator. Can be hidden via user preference.
