# UI Guidelines

General UI and interaction guidelines for koad:io products.

## Principles

- **Dark theme first:** koad:io ecosystem is terminal-native; dark theme is primary
- **Minimal decoration:** Beauty through clarity, not ornamentation
- **Consistency:** Visual and interaction patterns reused across products
- **Responsive:** Mobile-first; works at all breakpoints
- **Fast:** No bloat, fast loading, efficient repaints
- **Accessible:** WCAG 2.1 AA minimum on all products

## Key Guidelines

- **Layout:** Grid systems, spacing rules, breakpoint behavior
- **Typography:** Font pairing, sizing, hierarchy, readability
- **Color:** Palette usage, light/dark theme switching, semantic colors
- **Spacing:** Base unit (4px), margin/padding consistency, white space usage
- **Interactions:** Hover states, focus indicators, transitions, loading states
- **Mobile:** Touch targets, responsive text sizes, viewport-appropriate layouts
